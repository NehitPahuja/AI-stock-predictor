# 🚀 Quantum Ledger - Implementation To-Do List

This document provides a comprehensive, step-by-step plan to build **Quantum Ledger** (the AI Stock Predictor). It synthesizes the Machine Learning requirements from your `PRD` and `TECH` documents with the premium, dark-themed UI requirements from the provided reference images.

## Phase 1: Backend Foundation & Data Pipeline
*Objective: Set up the Python core and data fetching mechanisms as per the PRD/TECH.*
- [x] **1.1 Project Initialization:** Create the project directory structure (`data/`, `models/`, `src/`, `notebooks/`).
- [x] **1.2 Environment Setup:** Set up a Python virtual environment and `requirements.txt` (installing `yfinance`, `pandas`, `numpy`, `scikit-learn`, `xgboost`, `tensorflow`).
- [x] **1.3 Data Loader Module:** Implement `src/data_loader.py` to fetch historical stock data (US & Indian stocks) for free using `yfinance`.
- [ ] **1.4 Data Cleaning utilities:** Write functions to clean data, handle missing values, and perform forward/backward fills.

## Phase 2: Feature Engineering & MVP Models
*Objective: Build the core ML pipeline to process data and generate basic predictions.*
- [ ] **2.1 Feature Engineering (`src/features.py`):** Calculate Moving Averages (MA10, MA50), daily returns, and rolling volatility.
- [ ] **2.2 Baseline Model (`src/train.py`):** Train a Linear Regression model for next-day price prediction using the closing price.
- [ ] **2.3 Advanced MVP Model:** Train a Random Forest model incorporating the newly engineered features.
- [ ] **2.4 Prediction Engine (`src/predict.py`):** Implement the function to orchestrate the pipeline and format outputs (returning predicted values, confidence %, and bullish/bearish conviction) to match the UI requirements.

## Phase 3: Advanced ML & API Layer
*Objective: Implement the complex models defined in the PRD/TECH and expose them via a backend API.*
- [ ] **3.1 XGBoost Integration:** Add XGBoost for enhanced price/direction prediction.
- [ ] **3.2 LSTM Deep Learning:** Implement an LSTM model using TensorFlow for advanced sequence-based prediction (Phase 3 of PRD).
- [ ] **3.3 Model Evaluation:** Write validation scripts calculating MAE, RMSE, and Direction Accuracy score.
- [ ] **3.4 API Development (FastAPI or Flask):** Create a lightweight API wrapper around your Python ML tasks. 
  > [!TIP]
  > While the PRD mentions Streamlit, achieving the exact pixel-perfect aesthetics, animations, and complex custom dashboard layouts shown in the "Quantum Ledger" mockup is significantly easier and more flexible if you decouple the frontend (using HTML/Vanilla JS) from the ML backend.

## Phase 4: Frontend UI Construction (Quantum Ledger Dashboard)
*Objective: Set up the dark-mode, tactical frontend layout visible in the reference design.*
- [ ] **4.1 Layout Skeleton:** Build a responsive base layout featuring a locked left sidebar and a scrollable main content area.
- [ ] **4.2 Global CSS Architecture:** Define CSS variables for the dark aesthetic, primary colors (energetic greens, reds, electric blues), typography (e.g., modern fonts like Inter), and glassmorphism (translucency, backdrop-filters).
- [ ] **4.3 Sidebar Navigation:** Implement the left panel navigation menu (Terminal, Predictions, Sentiment, Portfolio, Settings) with active states, the user profile section, and the "Market Status" indicator.
